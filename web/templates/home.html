why it works this way

Built around one JSON file, not a black box

Local model, no API

The LLM runs on your hardware. No request ever leaves your machine, no rate limits, no per-token bill.

Definitions you control

Every callable function lives in one JSON file. Add one, the model can use it — no retraining, no fine-tuning.

Structured, not guessed

Constrained decoding keeps the model inside valid JSON — the output is always parseable, never a maybe.

the pipeline

Three steps, every time

1

You type a plain-language request

Something like what's the square root of 23 — no special syntax, no command format to memorize.

2

The model reads the function registry

It compares your prompt against the definitions in functions.json and decides which one — if any — fits.

3

The function runs, the result comes back

You see the exact function name, the arguments it extracted, and the value it returned — laid out, not buried in prose.
